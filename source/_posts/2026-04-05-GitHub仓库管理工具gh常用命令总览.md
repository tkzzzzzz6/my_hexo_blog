# GitHub 仓库管理工具 `gh` 常用命令总览

平时我们在命令行里管理 GitHub 仓库时，如果只靠网页操作，很多流程都会显得有点碎。比如查看 PR、创建 Issue、切分支、拉取远程仓库信息、检查登录状态，这些动作来回切页面确实容易把人搞烦。

这时候 `gh` 就很顺手了。它是 GitHub 官方提供的命令行工具，可以直接在终端里完成仓库、Issue、PR、Release、Actions 等常见操作。对于经常在服务器、WSL 或者终端开发环境里工作的同学来说，真的属于“装了就回不去”的那类工具。

这篇文章把我平时高频使用的 `gh` 命令做一个整理，方便自己回头查，也方便读者快速上手。

---

## 一、`gh` 是什么

`gh` 是 GitHub CLI，也就是 GitHub 官方维护的命令行工具。

它适合做的事情包括：

- 在终端中登录 GitHub 账号
- 克隆、查看和管理仓库
- 创建、查看和合并 Pull Request
- 创建、查看和关闭 Issue
- 查看 GitHub Actions 工作流执行情况
- 管理 Release、Gist 等资源

如果你的工作流本身就偏命令行，那 `gh` 基本可以把不少网页操作搬回终端。

## 二、安装与登录

### 1. 安装 `gh`

####  跨平台的安装(推荐)

直接使用x -cmd命令安装

```bash
x install gh
```

#### 不同平台下的安装
**Windows（winget）**

```powershell
winget install --id GitHub.cli
```

**Ubuntu / Debian**

```bash
sudo apt install gh
```

**macOS（Homebrew）**

```bash
brew install gh
```

安装完成后先检查版本：

```bash
gh --version
```

### 2. 登录 GitHub

首次使用通常先执行：

```bash
gh auth login
```

然后按照提示选择：

- GitHub.com 或企业版实例
- HTTPS 或 SSH
- 浏览器授权或 Token 授权

查看当前登录状态：

```bash
gh auth status
```

退出登录：

```bash
gh auth logout
```

## 三、仓库相关常用命令

### 1. 克隆仓库

```bash
gh repo clone owner/repo
```

例如：

```bash
gh repo clone cli/cli
```

如果想克隆到指定目录：

```bash
gh repo clone owner/repo my-local-dir
```

### 2. 查看仓库信息

```bash
gh repo view owner/repo
```

查看当前目录关联仓库的信息：

```bash
gh repo view
```

如果想直接在浏览器打开仓库：

```bash
gh repo view --web
```

### 3. 创建新仓库

```bash
gh repo create
```

也可以一步到位指定名称：

```bash
gh repo create my-demo --public
```

常见参数：

- `--public`：创建公开仓库
- `--private`：创建私有仓库
- `--clone`：创建后直接克隆到本地
- `--source .`：把当前目录作为仓库内容推送上去

### 4. Fork 仓库

```bash
gh repo fork owner/repo
```

如果希望 fork 后顺手 clone 到本地：

```bash
gh repo fork owner/repo --clone
```

### 5. 在浏览器中打开当前仓库

```bash
gh repo view --web
```

这个命令很适合“终端里改完，网页里看一眼”这种场景。

## 四、Issue 相关命令

### 1. 查看 Issue 列表

```bash
gh issue list
```

查看指定仓库的 Issue：

```bash
gh issue list --repo owner/repo
```

常见筛选：

```bash
gh issue list --state open
gh issue list --assignee @me
gh issue list --label bug
```

### 2. 查看某个 Issue

```bash
gh issue view 123
```

在浏览器中打开：

```bash
gh issue view 123 --web
```

### 3. 创建 Issue

```bash
gh issue create
```

直接在命令行中指定标题和内容：

```bash
gh issue create --title "fix: 修复登录页按钮样式" --body "按钮 hover 状态异常，需要统一样式。"
```

### 4. 关闭 Issue

```bash
gh issue close 123
```

如果要顺便留下关闭说明：

```bash
gh issue close 123 --comment "已在最新提交中修复。"
```

## 五、Pull Request 相关命令

### 1. 查看 PR 列表

```bash
gh pr list
```

常见筛选方式：

```bash
gh pr list --state open
gh pr list --author @me
gh pr list --assignee @me
```

### 2. 查看某个 PR

```bash
gh pr view 25
```

在浏览器打开：

```bash
gh pr view 25 --web
```

查看 PR diff：

```bash
gh pr diff 25
```

### 3. 创建 Pull Request

先确保你已经把本地分支推送到远程，然后执行：

```bash
gh pr create
```

也可以直接带参数：

```bash
gh pr create --title "feat: 新增文章搜索页" --body "补充搜索入口，并优化移动端布局。"
```

如果想指定 base 分支：

```bash
gh pr create --base main --head feature/search-page
```

### 4. 检出某个 PR 到本地

```bash
gh pr checkout 25
```

这个命令在帮别人 review 代码时特别方便，不需要自己手动找远程分支名。

### 5. 合并 PR

```bash
gh pr merge 25
```

常见合并方式：

```bash
gh pr merge 25 --merge
gh pr merge 25 --squash
gh pr merge 25 --rebase
```

### 6. 关闭 PR

```bash
gh pr close 25
```

## 六、工作流与 Actions 相关命令

### 1. 查看 workflow 运行记录

```bash
gh run list
```

### 2. 查看某次运行详情

```bash
gh run view RUN_ID
```

### 3. 查看日志

```bash
gh run view RUN_ID --log
```

### 4. 重新运行失败任务

```bash
gh run rerun RUN_ID
```

### 5. 在浏览器打开 Actions 页面

```bash
gh run list --web
```

如果你在维护博客自动部署、CI 检查或者测试流水线，这一组命令查问题很省时间。

---

## 七、Release 与其他实用命令

### 1. 查看 Release 列表

```bash
gh release list
```

### 2. 创建 Release

```bash
gh release create v1.0.0
```

带说明一起创建：

```bash
gh release create v1.0.0 --title "v1.0.0" --notes "首个正式版本发布。"
```

### 3. 下载 Release 附件

```bash
gh release download v1.0.0
```

### 4. 查看当前账号信息

```bash
gh api user
```

### 5. 打开当前目录对应网页

```bash
gh browse
```

这个命令非常偷懒，但很好用。

## 八、我自己最常用的一组组合命令

如果你只是想先记住最常用的那几个，可以先从下面这些开始：

```bash
# 登录与检查
gh auth login
gh auth status

# 仓库
gh repo clone owner/repo
gh repo view --web

# Issue
gh issue list
gh issue create

# Pull Request
gh pr list
gh pr create
gh pr view 12 --web
gh pr checkout 12
gh pr merge 12 --squash

# Actions
gh run list
gh run view RUN_ID --log
```

## 九、使用建议

`gh` 很强，但我建议别一开始就把所有子命令都背下来。先记住你当前工作流里最常用的 5 到 8 个命令就够了，比如：

- `gh auth login`
- `gh repo view --web`
- `gh issue list`
- `gh pr list`
- `gh pr create`
- `gh pr checkout`
- `gh run list`

等你开始习惯在终端里处理 GitHub 事务之后，再慢慢补充其他命令，会比一次性硬记舒服得多。

---

## 总结

`gh` 本质上就是把 GitHub 上那些高频但重复的网页操作，压缩成了一条条终端命令。对于经常写代码、提 PR、看 CI、管仓库的人来说，它能明显减少上下文切换的成本。

如果你平时已经离不开 `git`，那我建议顺手把 `gh` 也装上。很多时候你会发现，原来一个浏览器里点半天的流程，在终端里一句命令就结束了，确实挺香。
