**Training and evaluation details** 这部分主要介绍了 **MokA** 的**训练流程、模型配置和评估基准**。可以总结为以下四个核心要点：

---

## 1. 模型架构配置 (Implementation Details)

| 组件 | 具体配置 |
| :--- | :--- |
| **视觉编码器** | CLIP-ViT/L-14（提取最后一层 Patch 级特征） |
| **音频编码器** | BEATs（音频-视觉-文本场景） |
| **语音编码器** | OpenAI Whisper（语音-文本场景） |
| **Projector** | Q-Former + 两层 MLP（32 个 Query Token） |
| **LLM 骨干** | LLaMA-2/3, Qwen2, Qwen2.5-VL（全部冻结） |

**关键设定**：整个训练过程中，**LLM 权重始终保持冻结**，只有 Projector 和 MokA 模块参与训练。

---

## 2. 两阶段训练范式 (Two-Stage Training)

### **Stage 1: 预训练 (Pre-training)**
*   **目标**：跨模态对齐（让图像/音频特征与文本空间对齐）
*   **可训练参数**：仅 Projector
*   **数据集**：
    *   视觉：Video-LLaVA（视频/图像描述）
    *   音频：AudioCaps（音频描述）
    *   语音：GigaSpeech-M（语音识别）
*   **优化器**：AdamW，余弦学习率，初始 lr=1e-4，warmup=0.03

### **Stage 2: 指令微调 (Instruction-tuning)**
*   **目标**：下游任务适配（问答、推理）
*   **可训练参数**：Projector + **MokA 模块**
*   **数据集**：
    *   音视频-文本：MUSIC-AVQA, AVE
    *   视觉-文本：LLaVA-Instruct-150K
    *   语音-文本：LibriSpeech

---

## 3. 评估基准 (Evaluation Benchmarks)

| 场景 | 数据集 | 任务类型 |
| :--- | :--- | :--- |
| **音视频-文本** | MUSIC-AVQA, AVE | 时空推理、事件定位 |
| **视觉-文本** | MME, MMBench, POPE, SEED-Bench | 感知、推理、幻觉检测 |
| **语音-文本** | MMAU-mini-speech, AIR-Bench | 语音理解、识别 |

---

## 4. 核心实验设定

*   **对比基线**：LoRA, DoRA, LoRAMoE, HydraLoRA, Uni-modal LoRA 等
*   **关键超参**：
    *   LoRA 秩 $r$：通常设为 8
    *   $\lambda$（Cross-Attention 强度）：控制跨模态交互程度
    *   可训练参数占比：约 **1.33%**

---

## 一句话总结

> **MokA 采用"预训练对齐 + 指令微调适配"的两阶段策略，在冻结 LLM 骨干的前提下，通过训练 Projector 和 MokA 适配器，在多个多模态基准上实现了高效且精准的下游任务适配。**
